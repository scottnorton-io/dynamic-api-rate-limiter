# Security Policy

## Supported Versions

This project is currently in early development (`0.x` versions). Security
issues are still taken seriously, but the API may change without strict
backward compatibility guarantees.

## Reporting a Vulnerability

If you believe you have found a security vulnerability in this project:

1. **Do not** create a public GitHub issue describing the vulnerability in detail.
2. Instead, please contact the maintainer privately at:

   - Email: ideas@scottnorton.io

Please include:

- A description of the issue
- Steps to reproduce (if possible)
- Any potential impact you foresee

You can optionally include:

- Suggested mitigations or fixes
- Whether you are comfortable being acknowledged for the report

The maintainer will:

- Acknowledge receipt of your report as soon as possible.
- Investigate the issue and determine its impact.
- Prepare and publish a fix if required.
- Coordinate public disclosure timing with you when appropriate.
